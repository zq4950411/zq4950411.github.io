使用Hexo搭建博客

发表于 2015-12-24   |   分类于 建站   |     |   阅读次数

hexo是一个轻量级的博客工具，这篇文章只是对整个过程进行了高度概括，这有利于读者了解整个过程。如果中间有哪些不太清楚的地方，推荐阅读官方中文文档

文件目录

在指定文件夹创建Hexo博客文件

执行完上述命令，那么指定文件夹的目录结构如下

_config.yml

网站的所有配置都在这个文件中，你可以控制其中的内容来实现所需要的功能
鉴于配置文件功能强大，所以专门写了一篇文章，请点击这里

package.json

这里面存放的是插件信息，从这里可以看到安装了什么插件，内容如下：

安装的插件都存放在/node_modules文件夹中

scaffolds

这里面存放用户的模板。当您新建文章时，Hexo 会根据 scaffold 来建立文件。
这个主要是针对hexo new [layout] <title>命令。

  • 如果没有设置 [layout] 的话，默认使用 _config.yml 中的default_layout参数代替。
  • 如果写的[layout]不在这个文件夹里，那么还是会使用配置文件中默认的模板来创建文件。
  • <title>要用双引号。

source

这里面存放用户的资源文件。

  • 除 posts 文件夹之外，开头命名为 (下划线)的文件或文件夹和隐藏的文件将会被忽略。
  • Markdown 和 HTML 文件会被解析并放到 public 文件夹(最后的发布内容都在这里面存放)，而其他文件会被拷贝过去。

themes

这里没存放用户的主题。Hexo会根据主题来生成静态页面
在/_config.yml配置文件中修改以下信息来更换主题

操作指令

指令 描述 更多
hexo init [folder] 新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站
hexo new [layout] <title> 新建一篇文章。如果标题包含空格的话，请使用引号括起来。 hexo new post "haha"
hexo generate 生成静态文件，可以加参数 -d(–deploy),-w(–watch)
hexo publish [layout] <filename> 发表草稿
hexo server 启动服务器，可以加参数。默认情况下，访问网址为： http://localhost:4000/ -p(–port),-s(–static),-l(–log)
hexo deploy 部署网站，可加参数 -g(–generate)
hexo render <file1> [file2] ... 渲染文件,可以加参数(用于手动渲染指定文件) -o(–output)
hexo clean 清除缓存文件 (db.json) 和已生成的静态文件 (public)
hexo migrate <type> 从其他博客迁移内容
hexo list <type> 列出网站资料
hexo version 显示Hexo版本信息
hexo --debug 从终端中显示调试信息，并记录到debug.log,当遇到问题时可以尝试
hexo --safe 在安全模式下，不会载入插件和脚本。当您在安装新插件遭遇问题时，可以尝试以安全模式重新执行
hexo --silent 隐藏终端信息

更多指令请参考此处

布局(Layout)

Hexo 有三种默认布局：post、page 和 draft，它们分别对应不同的路径，而您自定义的其他布局和 post 相同，都将储存到 source/_posts 文件夹。

布局 路径 描述
post source/_posts 博客内容页面
page source 自定义文件，这个文件直接拷贝
draft source/_drafts 草稿布局

如果你不想你的文章被处理，你可以将 Front-Matter 中的layout: 设为 false 。

通过自定义布局文件(向文件中加入若干Front-matter)，可以省却很多时间

Front-matter

Front-matter 是文件最上方以 --- 分隔的区域，用于指定个别文件属性的变量。

以下是预先定义的参数，您可在模板中使用这些参数值并加以利用。

参数 描述 默认值
layout 布局 true
title 标题 :title
date 建立日期 文件建立日期
updated 更新日期 文件更新日期
comments 开启文章的评论功能 true
tags 标签 （不适用于分页）
categories 分类（不适用于分页）
permalink 覆盖文章网址

常见问题

文章每次修改一下自动跑到了列表最前端

默认情况下，hexo按照文章的修改时间来显示文章列表。所以，如果出现这个问题，只需要在Markdown文件中添加Front-matter，内容如下：

date以你想显示该文章的时间为主，建议为所有文章都加入该数据，以防日后修改以前文章时出现顺序不对的情况

这也是没有使用hexo new [layout] <title>命令的锅

如何绑定域名

在source文件夹里创建CNAME文件，不带任何后缀，里面添加你的域名信息，如：nstimer.cn（注意前面不添加http://）

两个不同Markdown文件之间进行链接

默认情况下，Markdown文件都被统一渲染为一个名称不变的文件夹，文件夹内部有一个固定的index.html的文件。所以我们引用的时候就要这么写才行。

(/2015/12/24/Hexo配置/index.html)

默认以时间进行排序，从而把时间写上就可以完成跳转了。这里的时间是该文章的date属性显示的时间
